# API Docs

This page shows the API reference by package first, then by user-interest surface.

The detailed package hubs below are the first stop. The member pages sit underneath them.

## Package split

The reference is split the same way the code is split:

- `FsFlow` for synchronous and async boundaries, validation helpers, and support types
- `FsFlow.Net` for task-oriented boundaries, cold task helpers, and task-native interop

## Read this first

Choose the package hub that matches the runtime shape of the code you are writing, then drill into the landing pages beneath it.
